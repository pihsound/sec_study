이메일 한 통, 어디를 거쳐 가나?
흐름 이해
나(홍길동)가 Outlook에서 김철수에게 이메일을 보낸다. 보내기를 클릭하는 순간 무슨 일이 벌어질까요?

이메일은 단순히 A에서 B로 날아가지 않아요. 마치 우체국 시스템처럼 여러 단계의 중간 처리 주체를 거쳐 전달됩니다. 이 중간 주체들이 바로 MUA · MSA · MTA · MDA · MS입니다.

이메일 전체 전송 흐름 — 송신자에서 수신자까지
한 줄 요약: MUA(작성) → SMTP → MSA(제출) → SMTP → MTA(전송) → 인터넷 → MTA(수신) → SMTP → MDA(배달) → MS(저장) → POP3/IMAP → MUA(읽기)

💡 핵심 암기 — SMTP는 "보내는" 전 구간에 쓰인다

SMTP는 MUA→MSA, MSA→MTA, MTA→MTA, MTA→MDA 전 구간에 사용돼요. 메일을 받아서 읽을 때만 POP3 또는 IMAP을 씁니다. "보내는 건 다 SMTP, 읽는 건 POP3/IMAP"으로 구분하세요.

4
POP3 vs IMAP — 수신 방식의 차이
비교
POP3 — Post Office Protocol 3
PORT : TCP 110
서버에서 메일을 다운로드한 후 삭제. 로컬에 저장됨. 오프라인 접근 가능. 여러 기기에서 쓰면 한 기기에서만 다운.
비유: 우체국에서 편지를 가져옴 → 우체국엔 없어짐
IMAP — Internet Message Access Protocol
PORT : TCP 143
메일이 서버에 유지됨. 여러 기기에서 동일한 메일함 접근 가능. 삭제·읽음 상태가 서버에 동기화됨.
비유: 우체국 편지함을 원격에서 열람 → 편지는 남아 있음
5
핵심 함정 — MSA와 MS 혼동
★★★ 기출
⚠함정 — "MSA가 POP3/IMAP으로 메시지를 추출한다" (2018 국가직 정보시스템보안)
② "MSA — 원격서버로부터 POP3 또는 IMAP를 사용하여 메시지를 추출한다" → ✗ 오답

POP3/IMAP으로 원격 서버에서 메시지를 추출하는 것은 MS(메시지 저장소)에 접근하는 MUA의 역할입니다. MSA는 MUA로부터 이메일을 받아 MTA로 전달하는 메일 제출 에이전트예요. MSA ≠ MS를 반드시 구별하세요.

정답 선지 확인: ①MUA=사용자 응용프로그램 ✓ / ③MDA=MHS에서 MS로 전달 ✓ / ④MTA=목적지 MDA까지 중계 ✓

⚠함정 — "MTA 역할이 뭐냐" (2024 국가직 정보시스템보안)
메일 클라이언트에서 ( )로 메일을 보낼 때와 ( )에서 ( )로 메일이 중계될 때 SMTP 프로토콜을 사용한다. → ✓ 정답 : MTA

MTA는 "메일 클라이언트(MUA)가 보내는 첫 서버"이자 "중계 서버(MTA↔MTA)"이기도 해요. 불특정 다수의 스팸 메일 전송·중계를 허용하면 공격자가 MTA를 스팸 도구로 악용할 수 있어요 → Open Relay 문제.

📄 2018 국가직 9급 · 정보시스템보안 문17 — 인터넷 메일 구조 오답★★★
인터넷 메일 구조의 핵심요소에 대한 설명으로 옳지 않은 것은?
① MUA — 사용자 액터(actor)와 사용자 응용프로그램을 대신하여 동작한다.
② MSA — 원격서버로부터 POP3 또는 IMAP를 사용하여 메시지를 추출한다.
③ MDA — 메시지를 MHS에서 MS로 메시지를 전달한다.
④ MTA — 메시지가 목적지 MDA에 도달할 때까지 중계 역할을 한다.
정답 ② — POP3/IMAP으로 원격 서버에서 메시지를 추출하는 것은 MUA가 MS에 접근하는 방식. MSA(Mail Submission Agent)는 MUA로부터 이메일을 접수하여 인터넷 MTA로 전달하는 메일 제출 에이전트.
📄 2024 국가직 9급 · 정보시스템보안 문9 — MTA 역할★★★
○ 메일 클라이언트에서 (가)로 메일을 보낼 때와 (가)에서 (가)로 메일이 중계될 때 SMTP 프로토콜을 사용한다.
○ 불특정 다수의 메일 전송 및 중계를 허용하면 공격자는 스팸 메일을 보내는 도구로 (가)를 악용할 수 있다.
① MTA (Message Transfer Agent)
② MDA (Mail Delivery Agent)
③ MUA (Mail User Agent)
④ MRA (Mail Relay Agent)
정답 ① — "클라이언트→(가)" + "(가)↔(가) 중계" + "SMTP" + "스팸 도구로 악용" = MTA. MTA는 메일을 라우팅·중계하는 메일 서버. Open Relay(열린 중계) 설정 시 스팸 서버로 악용됨.
📄 2021 군무원 7급 · 정보보호론 문18 — 이메일 보안 기술 구분★★
이메일 보안을 위해 사용되는 보안 기술로만 묶인 것으로 가장 옳은 것은?
① SET, S/MIME
② PEM, PGP
③ S/MIME, IDS
④ PGP, SSO
정답 ② — 전자우편 보안 기술: PGP·S/MIME·PEM (3가지). SET은 전자상거래 보안, IDS는 침입탐지시스템, SSO는 단일 로그인. ①의 SET·③의 IDS·④의 SSO는 이메일 보안이 아님.
📄 2019 국가직 정보시스템보안 문5 — PGP 특징★★★
전자우편의 암호화에 대한 설명이다. 괄호 안에 들어갈 용어는?

( )은/는 IDEA 알고리즘과 RSA 알고리즘을 조합하여 만들었다. IDEA 알고리즘은 세션키 암호화, RSA 알고리즘은 사용자 인증을 위한 전자서명에 이용하였다. 이것의 장점으로는 구현이 쉽고, 특정 기관으로부터 인증서를 발급받지 않아도 된다는 것이다.
① PGP
② PEM
③ S/MIME
④ IMAP
정답 ① — "IDEA + RSA" + "인증서 불필요" + "CA 없음" = PGP. PEM은 IETF 표준 Base64 방식. S/MIME은 X.509 인증서 필요. IMAP는 수신 프로토콜(143번 포트)로 이메일 보안 프로토콜이 아님.
