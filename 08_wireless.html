1
무선 LAN(Wireless LAN) 정의 및 특징
★★
무선 LAN 정의

무선 LAN(Wireless LAN)은 케이블 없이 전파를 이용할 수 있게 해주는 무선 인터넷을 의미하며, 흔히 Wi-Fi라고도 한다. Wi-Fi(Wireless Fidelity)는 국제 인증 기관인 Wi-Fi Alliance가 IEEE 802.11x를 지원하여 상호 관련됨을 인증한 표준 규격. 2.4~5GHz 무허가 주파수 대역을 활용. 기존 10m 거리에서 200~300m 대폭 증가하여 멀티미디어 서비스 수신 가능.

무선 LAN은 유선 LAN에 비해 편리하지만 보안 위협이 훨씬 크요. 유선은 물리적으로 선을 연결해야 하지만, 무선은 공기 중으로 데이터가 흘러다녀요. 안테나만 있으면 누구든 신호를 수신할 수 있어요. 이 때문에 도청, 불법 접속, 중간자 공격, Rogue AP 등에 취약해요.

무선 LAN의 주요 특징: ① 물리적 이동성(기기를 들고 이동하면서 사용 가능) ② 무허가 주파수 대역 사용 ③ 전파 범위 내 누구나 신호 수신 가능 ④ 유선에 비해 일반적으로 품질이 낮음 ⑤ 전파 간섭 존재.

CSMA/CA — 무선 LAN 매체 접근 방식
★★ 심화
CSMA/CA 정의

CSMA/CA(Carrier Sense Multiple Access / Collision Avoidance)는 IEEE 802.11 무선 LAN이 경로 공유를 위해 모두 사용하는 인터넷 프로토콜. 무선 LAN(IEEE 802.11)에서 사용되는 접근 방식. 데이터의 흐름을 감지하여 네트워크가 사용 중이 아니면 바로 패킷을 보내지 않고, 예비 신호를 먼저 보내 데이터 전송 및 패킷 충돌(Collision)을 피한다. CSMA/CA의 Multiple Access와 Carrier Sense는 CSMA/CD에도 적용된다.

유선 LAN에서는 CSMA/CD(Collision Detection — 충돌 감지)를 사용하지만, 무선 LAN에서는 CSMA/CA(Collision Avoidance — 충돌 회피)를 사용해요. 무선 환경에서는 충돌을 감지하기 어렵기 때문에, 미리 예비 신호를 보내 다른 기기의 전송을 기다린 후 데이터를 전송하는 방식이에요.

무선 LAN 주요 취약점 분류
★★★

무선 LAN의 취약점은 크게 비인가 접속, 도청·스니핑, 서비스 거부(DoS), 불법 AP 4가지로 분류해요. 유선 LAN과 달리 물리적 접근 없이도 공격이 가능해서 훨씬 취약해요.

취약점 유형원인내용대응 방안
① SSID 노출 AP가 SSID를 브로드캐스트하도록 기본 설정 SSID(Service Set Identification) = 무선 네트워크를 위한 이름. 일반적인 무선 AP의 경우 SSID를 브로드캐스트하도록 설정되어 있어 스니핑 등을 통해 쉽게 SSID를 획득할 수 있음. SSID를 알면 인증이나 암호화 없이도 필요한 기본적인 네트워크 정보 획득 가능. SSID 숨김(Hidden) 설정으로 브로드캐스트 차단 → 폐쇄시스템 운영. 단, 완전한 보안 아님.
② MAC 주소 노출 무선 프레임 헤더에 MAC 주소가 평문으로 전송 무선 통신 중 MAC 주소가 노출됨. MAC 주소 필터링 적용 시 정상 MAC을 스니핑으로 획득하여 MAC을 변조(MAC 스푸핑)하면 필터링 무력화 가능. MAC 필터링 + 강력한 암호화 병행. 단, MAC 스푸핑에 취약.
③ 도청·스니핑 무선 신호가 공기 중에 노출 암호화되지 않은 무선 통신은 전파 범위 내 누구나 수신 가능. WEP는 수분 내 해독 가능. 암호화되지 않는 모든 정보가 바로 수집 가능. 강력한 암호화(WPA2-AES) 필수 적용.
④ 불법 AP (Rogue AP) 비인가 AP를 공격자가 설치 외부 공격자가 악의적으로 무선 AP를 설치하여 이용자의 데이터 전송을 수집. 회사 내의 보안 정책을 따르지 않은 설치된 액세스 포인트. 가능한 신호를 가지고 AP를 설치하는 것이 어렵지 않아 사용자의 정상 AP 접속을 방해하는 것이 어렵다. 무선 침입탐지시스템(WIDS) 운영. AP 목록 주기적 점검.
⑤ Evil Twin 정상 AP로 위장한 악성 AP 설치 정상 AP처럼 위장하기 위해 동일 SSID로 설치된 무선 AP. 사용자가 Evil Twin에 접속하면 MITM(중간자) 공격 가능. Rogue AP의 특수한 형태. 인증서 기반 AP 검증. VPN 사용.
⑥ 서비스 거부 (DoS) 전파 방해 또는 인증 프레임 공격 무선 AP 또는 무선 네트워크를 공격하는 서비스 거부. 무선 AP가 사용하는 주파수 대역에 대해 강한 신호를 전송하여 서비스를 무력화. 전파 방해(Jamming) 공격. 주파수 모니터링. 채널 변경. 지향성 안테나 사용.
⑦ WEP 암호 취약점 RC4·IV 설계 결함 짧은 초기 벡터(IV, 24비트)로 인한 IV 재사용 가능성. 불완전한 RC4 암호화로 키 노출. 짧은 키 길이로 인한 공격 가능성 증가. 무선 전송 노출 위험 높음. WEP 사용 금지 → WPA2로 업그레이드.
⑧ WPA/WPA2 취약점 (PSK) PSK 방식에서 4-way 핸드셰이크 노출 PSK 방식 중 초기에는 PSK를 이용해 4-way Handshaking을 통해 무선 AP와 동일한 키를 갖고 있는지 확인하는 과정에서 암호화되어 있지 않아 무선 패킷 수집 후 분석하는 것이 가능. KRACK 공격 취약점. 기업 모드(802.1x/EAP) 사용. KRACK 패치 적용.

무선 LAN 개요
★★
무선 LAN 정의

무선 LAN(Wireless LAN)은 케이블 없이 전파를 이용할 수 있게 해주는 무선 인터넷을 의미하며, 흔히 Wi-Fi라고도 한다. Wi-Fi(Wireless Fidelity)는 국제 인증 기관인 WiFi Alliance가 IEEE 802.11x를 지원하여 상호 관련됨을 인증한 표준 규격. 2.4~5GHz 주파수 대역을 사용하며 무허가 주파수 대역을 활용하는 기술이다.

무선 LAN은 편리하지만 유선 LAN보다 보안 위협이 훨씬 크요. 유선은 물리적으로 선을 연결해야 하지만, 무선은 공기 중으로 데이터가 흘러다녀요. 안테나만 있으면 누구든 데이터를 수신할 수 있어요. 이 특성 때문에 도청·무단 접속·중간자 공격에 훨씬 취약해요.

IEEE 802.11i는 IEEE 802.11 무선 LAN의 보안 표준으로, 인증·데이터 무결성·데이터 기밀성에 관한 규격을 다루어요. 이 표준이 진화하는 과정에서 Wi-Fi Alliance가 WPA와 WPA2를 공표했어요.

WEP (Wired Equivalent Privacy)
★★★
WEP 정의

WEP(Wired Equivalent Privacy)는 초기 무선랜 보안설정 방법으로 유선랜과 동등한 수준의 보안성 제공을 목적으로 만들어진 보안 기술. 대칭키 기반 암호화 방식으로 현재는 암호 알고리즘 자체의 취약성이 많이 알려져 있어 사용이 권고되지 않음. RC4 알고리즘 사용. 인증: PSK(Pre-Shared Key). 암호화: RC4(64bit/128bit).

2
WEP 암호화 동작 원리
원리 이해

WEP의 암호화 과정을 단계별로 이해할게요. ① PRNG(의사난수 생성기)에 초기 벡터(IV, Initialization Vector)와 WEP 키를 입력하여 키스트림(Keystream)을 생성해요. (RC4 알고리즘) ② 키스트림과 전송 데이터를 XOR하여 암호문을 생성해요. ③ 생성된 암호문에 초기 벡터(IV)를 추가하여 전송해요.

하나의 키스트림(keystream)을 사용하여 전송 데이터를 암호화하는 것은 아주 쉽게 데이터가 노출될 수 있으므로, 전송되는 패킷마다 서로 다른 값의 키스트림을 이용하여 암호화하는 방식을 사용해요. 그런데 문제는 IV가 24비트로 짧아 2²⁴(약 1600만)번 사용 후 반복되며, 같은 IV가 반복되면 분석이 가능해져요.

3
WEP 취약점
★★★
① IV 재사용
초기 벡터(IV)가 24비트로 짧음. 약 5,000~5,000만 패킷 후 IV가 재사용되어 같은 IV에서 생성된 키스트림이 반복 → 통계적 분석 가능. 64비트 WEP 키는 수분 내 해독 가능.
② 고정 키 문제
WEP 키가 고정(정적)이에요. 키를 한 번 설정하면 바꾸지 않는 경우 많음. 키가 하나 노출되면 전체 통신이 해독됨.
③ 단방향 인증
단방향 인증으로 인한 악의적인 목적으로 운영되는 복제 AP(Clone AP)를 통해 정상 AP로의 접속을 방해하거나 정상 사용자의 복제 AP로의 접속을 유도할 수 있음.
④ RC4 취약성
RC4 스트림 암호 알고리즘 자체에 알려진 약점 존재. WEP키를 하나의 고정된 공유키로 사용하여 키값이 외부로 유출될 경우 많은 보안 문제를 유발할 수 있음.
1
WPA (WiFi Protected Access)
★★★
WPA 정의

WPA(WiFi Protected Access)는 IEEE 802.11i의 표준화 지연으로 WEP의 취약성을 보완하고자 WiFi Alliance에서 정한 인증 및 암호화 관련 무선랜 보안 규격. 802.1x 인증과 TKIP을 수용하며 802.11i의 Subset 표준. WPA는 IEEE 802.11i Draft 3.0의 Subset이며, 새로운 장비의 교체 및 구매 없이 기존 H/W에 S/W의 업그레이드로 적용 가능. 인증: PSK(개인모드) 또는 IEEE 802.1x/EAP(기업모드). 암호화: RC4-TKIP.

WPA의 가장 큰 특징은 TKIP(Temporal Key Integrity Protocol)이에요. WEP의 고정 키 문제를 해결하기 위해 패킷마다 키를 동적으로 변경해요. 키 할당, 재설정 등 기존 WEP 소프트웨어 패치만으로 안전성을 개선한 보안 기술이에요. 그러나 RC4를 여전히 사용한다는 한계가 있어요.

비인가 AP로의 연결을 방지하기 위해 상호인증(Mutual Authentication)으로 구현돼요. AP와 사용자 사이에는 Open System 인증을 지원해야 해요.

2
TKIP (Temporal Key Integrity Protocol)
★★★
TKIP 정의

TKIP(Temporal Key Integrity Protocol)는 WEP 알고리즘의 취약성을 보완하기 위해 연구된 기술. 패킷당 키 할당, 키 재설정 등 기존 WEP 방식에 소프트웨어 패치만으로 안전성을 개선. 암호키가 각 데이터 프레임마다 변경되도록 만들고 메시지 무결성을 보장하기 위해 MIC(Message Integrity Code)를 프레임에 포함.

3
WPA2 (Wi-Fi Protected Access 2)
★★★
WPA2 정의

WPA2(Wi-Fi Protected Access 2)는 802.11i 보안 표준 중 일부분으로 WEP 방식의 보안 문제점을 해결하기 위해 만들었음. WPA와 구분하기 위해 WPA2로 불린다. AES-CCMP를 통한 암호화 기능 향상, EAP 사용자 인증 강화. 암호화 알고리즘으로 AES를 사용하며 WEP보다 훨씬 더 강화된 암호화 서비스를 제공. WPA2에서는 CCMP가 강제사항(Mandatory)으로 규정되어 있음.

WPA2의 핵심은 CCMP(Counter Mode With Cipher Block Chaining Message Authentication Code Protocol)예요. AES를 이용한 카운터 모드로 동작하며 기밀성·메시지 무결성·키 관리 기능을 모두 수행해요. 무선 보안표준인 IEEE 802.11i와 WPA에서 공동채택됐고, WPA2에서는 강제사항으로 규정됐어요.

암호화 키를 이용해 128비트인 통신용 암호화 키를 새로 생성하고 10,000 패킷마다 바꿔요. AP에 접속하는 사용자마다 같은 암호화 키를 사용하는 보안 취약성이 아직 존재하지만, 현재 가장 강력한 무선 LAN 보안 기술이에요.

4
CCMP (Counter Mode with CBC-MAC Protocol)
★★★
CCMP 정의

CCMP(Counter Mode With Cipher Block Chaining Message Authentication Code Protocol)는 AES를 이용한 블록암호의 카운터 모드. 기밀성·메시지 무결성·키 관리를 행한다. 무선 보안표준인 IEEE 802.11i와 WPA에서 공동채택. WPA2에서는 강제사항으로 규정되어 있음.

802.1x / EAP — 기업용 인증
★★★
802.1x 정의

IEEE 802.1x는 유선망의 표준을 무선랜으로 확장하며 무선 802.11 네트워크에 대한 인증된 Network Access를 제공하는 IEEE 표준. 중앙 사용자 ID·인증·동적 키 관리 및 계정을 지원하는 무선랜 인증 및 암호화 표준 기술. Port Based Access Control로 AP에 접속 요청이 오면 먼저 인증장비를 통해 인증을 요청하고, 인증이 완료되면 접속을 허용.

IEEE 802.1x는 RADIUS 서버를 이용해 중앙 집중식으로 사용자를 인증해요. EAPOL(EAP Over LAN) 패킷이 클라이언트와 AP 사이에서 전송되고, AP와 인증 서버 사이에는 RADIUS를 통해 전달해요.

EAP 정의

EAP(Extensible Authentication Protocol)는 IEEE 802.1x에서 사용자 인증을 위해 사용하는 프로토콜. 확장 인증 프로토콜로 RFC 2284에서 공식적인 표준이 되었으며, 원래는 PPP(Point to Point Protocol)에서의 인증을 위해 발전됐으나 현재는 IEEE 802.1x의 사용자 인증으로 사용. 어떤 링크에도 가능한 단순한 Request/Response 프로토콜로 다양한 인증 방법 사용 가능: MD5, 일회용 패스워드, 토큰카드, 인증서 기반 EAP-TLS, 터널로 보호된 TLS인 EAP-TTLS 등.

802.1x의 단점: IEEE 802.1x 규격은 무선구간의 보안을 위한 키 분배 시점 및 키 분배 여부를 사용자 인증에 참조하는 조건을 정의하지 않아 인증과 키 분배라는 암호학적 프로토콜의 주요기능을 만족하지 못하는 취약점을 내포한다고 알려져 있어요.

2
SSID / MAC 주소 필터링 — 기술적 보안 방안
★★
보안 방안원리취약점
SSID 설정 AP가 사용하는 ID를 통해 적재된 패킷 헤더에 붙는 32바이트 고유 식별자. AP에서 설정된 SSID와 일치하는 무선 단말기만 접속 허용. 스니핑으로 쉽게 SSID를 알아낼 수 있음. SSID를 알면 쉽게 정상 AP를 통해 사용할 수 있는 이점이 있어 보안 효과 제한적.
MAC 주소 필터링 MAC = 네트워크 카드에 의해 부여된 48비트 물리적 주소. AP가 고유 MAC 주소를 등록하여 MAC 주소 미등록 단말기의 접속을 차단. MAC 스푸핑(MAC Spoofing): MAC 주소를 다른 MAC 주소로 위장하는 것이 가능. AP에 등록된 MAC 주소를 알아내고 자신의 MAC 주소를 바꾸면 MAC 필터링이 무력화됨.
WEP 인증 공개키 WEP 키를 이용하여 사용자를 인증. AP를 이용한 WEP 인증 및 인증서버를 이용한 인증 모두 AP 또는 인증 서버와 사용자의 단말기에 같은 값의 WEP 키를 설정하여 사용. RC4 취약성, 64비트 WEP 키는 수분 내 노출 가능. 취약하여 널리 사용되지 않음.
3
KRACK 공격 (심화 — 1~2회 출제)
★★ 2020 경찰간부
KRACK 정의

KRACK(Key Reinstallation Attacks)은 WPA2의 4-way 핸드셰이크(Handshake) 과정에서 메시지를 조작하고 재전송하여 정보를 획득하는 공격. 2017년 발견. 이미 사용된 암호화 키를 재설치(Reinstallation)하도록 유도하여 암호화된 통신을 복호화·위조·주입할 수 있음. WPA2의 프로토콜 설계상 취약점으로 특정 구현체가 아닌 표준 자체의 취약점.

1
블루투스(Bluetooth) 보안 위협 5종
★★★ 매년 출제

블루투스 보안 공격은 이름이 비슷해 혼동하기 쉬워요. 정확한 정의와 키워드를 매핑해서 암기해야 해요. 블루재킹 → 스팸 메시지, 블루스나프 → OPP/파일 접근, 블루버그 → 임의 명령 실행, 블루프린팅 → 기기 정보 수집(SDP), BIAS → 인증 취약점 위장 순으로 구분하면 돼요.

📩 블루재킹 (Bluejacking)
정의: 블루투스를 이용해 스팸처럼 명함을 익명으로 퍼트리는 공격. 비요청 메시지·이미지를 전송.
특징: 데이터의 이동이나 변조가 없어 사용자를 귀찮게 하는 문제 외에는 별다른 위험 없음. 가장 낮은 위협 수준.
키워드: 스팸, 명함, 익명 메시지, 별다른 위험 없음
📂 블루스나프 (Bluesnarf / Bluesnarfing)
정의: 블루투스 장치끼리 인증 없이 정보를 간편하게 교환하기 위해 개발된 OPP(OBEX Push Profile) 기능을 사용하여 블루투스 장치로부터 주소록·달력 등의 내용을 요청해 이를 열람하거나 취약한 장치의 파일에 접근하는 공격.
키워드: OPP, OBEX Push Profile, 주소록·달력 열람, 파일 접근
📱 블루버그 (Bluebug / Bluebugging)
정의: 블루투스 장비 간의 취약한 연결 관리를 악용한 공격. 공격 장치와 공격 대상 장치를 연결하여 공격 대상 장치에서 임의의 동작(전화 걸기·다른 기능)을 실행하도록 하는 공격. 블루투스 기기가 한 번 연결된 이후에는 다시 연결하지 않아도 자동으로 연결되는 인증 취약점을 이용.
키워드: 취약한 연결 관리, 임의 동작 실행, 자동 연결, 전화 걸기
🔍 블루프린팅 (Blueprinting)
정의: 블루투스 장치는 장치 간 종류 식별을 위해 SDP(Service Discovery Protocol)를 보내고 받음. 이 서비스 발견 프로토콜을 이용해 공격이 가능한 블루투스 장치를 검색하고 모델을 확인. 블루투스 공격 장치의 검색 활동.
키워드: SDP, 서비스 발견 프로토콜, 기기 검색·정보 수집, 모델 확인

② ㉠802.11i ㉡DES ㉢TKIP ㉣RADIUS
③ ㉠802.1x ㉡DES ㉢TKIP ㉣EAP
④ ㉠802.1x ㉡TKIP ㉢AES ㉣RADIUS

① ㉠WPA ㉡Wi-Fi Alliance ㉢WPA2
② ㉠WPA ㉡IETF ㉢WPA2
③ ㉠WEP ㉡Wi-Fi Alliance ㉢WPA
④ ㉠WEP ㉡IETF ㉢WPA
정답 ③ — ㉠ 무선 랜 초기 보안 = WEP. ㉡ WPA를 공표한 주체 = Wi-Fi Alliance (IETF 아님). ㉢ 802.11i Draft 기반 중간 조치 = WPA. 이후 IEEE 802.11i 완전 표준 = WPA2.
📄 2017 국회직 9급 · Wi-Fi 보안 기술 오답★★★
와이파이(Wi-Fi) 보안 기술에 대한 설명으로 옳지 않은 것은?
① IEEE 802.11 표준 기반의 무선 랜 기술이다.
② WEP 방식은 현재 보안상 취약점이 발견되었다.
③ WEP 방식은 MAC(Media Access Control) 주소 인증 프로토콜을 사용한다.
④ WPA 방식은 TKIP(Temporal Key Integrity Protocol)를 사용한다.
⑤ WPA2 방식은 AES-CCMP(Counter Mode CBC-MAC Protocol)를 사용한다.
정답 ③ — WEP는 PSK(Pre-Shared Key) 인증 방식이에요. MAC 주소 인증은 WEP의 인증 방식이 아니라 별도의 MAC 주소 필터링 보안 기법. ④WPA=TKIP, ⑤WPA2=AES-CCMP는 모두 맞아요.
📄 2020 서울시 7급 · WEP→TKIP→CCMP, MIC 빈칸★★★
IEEE 802.11i 표준은 무선랜 사용자 보호를 위해 사용자 인증 방식, 키 교환 방식 및 향상된 무선구간 암호 알고리즘을 정의하고 있다. 무선구간에서 전송되는 데이터를 보호하기 위한 방법으로 (㈎)의 취약성을 해결한 (㈏) 방식과 AES 암호 알고리즘을 사용하는 (㈐) 방식을 지원한다. (㈏) 방식은 암호키가 각 데이터프레임마다 변경되도록 만들고 메시지 무결성을 보장하기 위해 (㈑)을 프레임에 포함시킴으로써 (㈎)의 보안 취약성을 해결하였다.
① ㈎WEP ㈏TKIP ㈐CCMP ㈑MIC
② ㈎WAP ㈏TKIP ㈐CCMP ㈑MAC
③ ㈎WEP ㈏CCMP ㈐TKIP ㈑MAC
④ ㈎WAP ㈏CCMP ㈐TKIP ㈑MIC
정답 ① — ㈎ 취약성 해결 대상 = WEP. ㈏ WEP를 해결한 방식 = TKIP. ㈐ AES 사용 방식 = CCMP. ㈑ 메시지 무결성 보장 위해 프레임에 포함 = MIC(Message Integrity Code). MAC는 미디어 접근 제어로 다른 개념.
