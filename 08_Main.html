1
XSS (Cross-Site Scripting)
★★★ 최빈출
XSS 정의

공격자가 웹 취약점이 있는 서버에 악성 스크립트를 삽입하고, 이 스크립트가 다른 사용자의 브라우저에서 실행되도록 하는 공격. 서버가 입력값을 검증 없이 그대로 HTML에 포함시킬 때 발생. 사용자의 세션 탈취, 웹사이트 변조, 악의적 사이트로 이동이 가능하다.

XSS 공격 흐름 — Stored XSS (저장형)
XSS 공격 순서 (기출 정답): 라(서버에 XSS 코드 저장) → 가(사용자 접근) → 다(서버가 XSS 포함 응답 전달) → 나(사용자 브라우저에서 실행) → 마(결과 공격자에게 전달) — 2020 국가직 정보시스템보안 문20 정답 ③
XSS 유형특징공격 방식
Stored XSS (저장형)가장 일반적. 서버 DB에 저장됨게시판에 악성 스크립트 삽입 → 다른 사용자가 읽을 때 실행
Reflected XSS (반사형)URL 파라미터에 스크립트 삽입악성 URL을 이메일 등으로 배포 → 클릭 시 즉시 실행
DOM Based XSS서버 응답 없이 브라우저에서만 실행DOM 조작으로 스크립트 실행. 서버 로그에 안 남음
XSS 대응 방법

출력 인코딩< → < / > → > / " → "
보안 헤더CSP(Content Security Policy) 설정
HttpOnly쿠키에 HttpOnly 플래그 설정 (JS 접근 차단)
XSS 공격 목적
세션 탈취쿠키 훔쳐 로그인 세션 도용
페이지 변조웹 사이트 내용 무단 변경

CSRF (Cross-Site Request Forgery)
★★★ 최빈출
CSRF 정의

로그인한 사용자가 자신의 의지와 무관하게 공격자가 의도한 행위(수정·삭제·등록·송금 등)를 특정 웹사이트에 요청하게 만드는 공격. 웹사이트가 사용자의 브라우저(세션·쿠키)를 신뢰하는 점을 악용. XSS와 유사하나 악성 스크립트 실행 위치가 다름.

3
SQL Injection
★★★
SQL Injection 정의

공격자가 입력 폼에 특수한 SQL 코드를 삽입하여 DB 인증을 우회하거나 데이터를 조회·수정·삭제하는 공격. 웹 서버가 사용자 입력을 검증 없이 SQL 쿼리에 포함할 때 발생. OWASP Top 10에 매년 포함되는 핵심 취약점.

SQL Injection 동작 원리
대응 ① Prepared Statement
바인딩 변수 사용: WHERE ID=? AND PW=? — 입력값이 SQL 쿼리로 해석되지 않음
대응 ② 특수문자 필터링
' " / ; -- + 공백 등 SQL 예약어·특수문자 포함 시 차단. 입력값 화이트리스트 검증.
주의 — SQL vs 다른 공격
SQL Injection은 DB 조작이 목적. XSS는 스크립트 실행이 목적. 혼동 함정 주의!

HTTPS 완전 정리
★★★
HTTPS 정의

HTTPS = HTTP over SSL/TLS. 웹브라우저와 웹서버 간의 안전한 통신을 위해 HTTP에 SSL을 결합한 프로토콜. RFC 2818 문서에 정의. 정상 HTTP는 80번 포트 사용, HTTPS 지정 시 443번 포트 사용하며 SSL을 호출한다.

HTTPS 암호화 시 보호되는 요소 vs 보호 안 되는 요소
HTTP 포트
기본 TCP 80번. HTTP/3부터 UDP 사용. 상태를 저장하지 않는 Stateless 프로토콜.
HTTPS 포트
기본 TCP 443번. HTTP + SSL/TLS 결합. URL에 https:// 사용.
RFC
HTTP 1.0=RFC 1945 / HTTP 1.1=RFC 2616 / HTTPS=RFC 2818
2
쿠키 (Cookie)
★★
쿠키 정의

쿠키는 1994년 넷스케이프에서 처음 사용. 사용자가 웹사이트를 방문할 때 생기는 4KB 이하의 텍스트 파일. 클라이언트 컴퓨터에 저장됨. 사용자와 웹사이트를 연결해주는 정보 포함. 다음 방문 시 서버로 전송하여 클라이언트 신원 확인에 활용.

쿠키 특성내용
저장 위치클라이언트 컴퓨터 (서버가 아님)
용도사이트 개인화 / 장바구니 시스템 / 웹사이트 이용 방식 추적
바이러스 전파직접 바이러스 옮기거나 악성코드 설치 불가
저장 내용 ✗ 오답 주의"로그인·로그아웃·시스템 재부팅 정보 저장" → ✗ 오답 (2020 국가직 정보시스템보안)
보안 위협XSS 공격으로 쿠키 탈취 가능 → 세션 하이재킹으로 이어짐
대응HttpOnly 플래그 (JS 접근 차단) / Secure 플래그 (HTTPS 전용)
⚠쿠키 함정 — "로그아웃·재부팅 정보 저장" (2020 국가직 정보시스템보안)
④ "사용자들의 로그인, 로그아웃, 시스템 재부팅 정보를 저장한다" → ✗ 오답

쿠키는 사이트 개인화·장바구니·이용 방식 추적에 사용. 로그아웃·재부팅 정보는 시스템 로그(wtmp, utmp)에 저장됨. 쿠키가 바이러스를 옮기지 않는다는 것도 함께 암기.

PGP (Pretty Good Privacy)
★★★
PGP 정의

필 짐머만(Phil Zimmermann)이 1991년 제안한 전자우편 보안 프로토콜. 무료로 제공됨. 인증기관(CA) 없이 공개키 인증을 사용자 개개인에게 위임 (Web of Trust / 분산 신뢰 모델). IDEA·AES·CAST·3-DES 대칭 암호 사용. 기밀성·인증·무결성·부인방지 제공.

2
S/MIME (Secure/MIME)
★★★
S/MIME 정의

RSA 데이터 보안이 MIME 인터넷 메일 형식을 보안 강화시킨 것. IETF 작업 그룹에서 표준화. X.509 버전 3 인증서 사용 (PGP와의 차이점). 기밀성·무결성·사용자 인증·송신 사실 부인 방지 제공. 암호화 알고리즘: 메시지=Triple-DES, 서명=DSA, 해시=SHA-1.

3
PEM (Privacy Enhanced Mail)
보충
PEM 정의
인터넷 표준안으로 IETF에서 만든 암호화 기법. 이진 데이터를 6비트씩 분할하여 아스키 변환하는 Base64 사용.
특징
자동 암호화로 전송 중 유출돼도 내용 확인 불가. PGP보다 보안능력 뛰어남. 중앙 집중식 키 인증방식으로 대중화 어려움.

PGP의 최대 특징은 CA 없는 분산 신뢰 모델. 공개키 인증을 사용자 개개인에게 위임. S/MIME이 X.509 인증서·CA를 사용하는 것과 반대. "PGP는 구현이 쉽고, 특정 기관으로부터 인증서를 발급받지 않아도 된다" → ✓ 정답 (2019 국가직 정보시스템보안)

기출문제 전수 해설 12문제
연도순
📄 2014 서울시 9급 · XSS 설명 매칭★★★
웹사이트에서 입력을 엄밀하게 검증하지 않는 취약점을 이용하는 공격으로, 사용자로 위장한 공격자가 웹사이트에 프로그램 코드를 삽입하여 나중에 이 사이트를 방문하는 다른 사용자의 웹 브라우저에서 해당 코드가 실행되도록 한다. 이 공격은?
① HTTP 세션 탈취 (Session Hijacking)
② 피싱 (Phishing)
③ 사이트 간 스크립팅 (XSS)
④ 파밍 (Pharming)
정답 ③ — "입력 미검증 + 코드 삽입 + 다른 사용자 브라우저에서 실행" = XSS. 세션 하이재킹은 TCP 시퀀스번호 공격, 피싱은 가짜 사이트 유도, 파밍은 DNS 조작.
📄 2015 국가직 9급 · SQL Injection 설명 매칭★★★
공격자가 사용자의 명령어나 질의어에 특정한 코드를 삽입하여 DB 인증을 우회하거나 데이터를 조작한다. 이 공격은?
① 직접 객체 참조
② Cross Site Request Forgery
③ Cross Site Scripting
④ SQL Injection
정답 ④ — "명령어·질의어에 코드 삽입 + DB 인증 우회 + 데이터 조작" = SQL Injection. XSS는 스크립트 삽입, CSRF는 강제 요청이 핵심.
📄 2016 정보시스템보안 · CSRF 설명 매칭★★★
( )는 불특정 다수를 대상으로 로그인된 사용자가 자신의 의도와는 무관하게 공격자가 의도한 행위(수정, 삭제, 등록, 송금)를 하게 만드는 공격이다.
① 명령 삽입 취약점
② XSS 취약점
③ 디렉터리 리스팅 취약점
④ CSRF 취약점
정답 ④ — "불특정 다수 + 로그인 사용자 + 의도와 무관한 행위 강제" = CSRF. 핵심 키워드를 외울 것: "의지와 무관하게" + "웹사이트가 사용자 브라우저를 신뢰".
📄 2019 국가직 정보시스템보안 문5 · PGP 설명★★★
전자우편 암호화에 대한 설명이다. 괄호 안에 들어갈 용어는?
( )은/는 IDEA 알고리즘과 RSA 알고리즘을 조합하여 만들었다. 장점으로는 구현이 쉽고, 특정 기관으로부터 인증서를 발급받지 않아도 된다는 것이다.
① PGP
② PEM
③ S/MIME
④ IMAP
정답 ① — "IDEA(대칭) + RSA(공개키) 조합 + 인증서 불필요" = PGP. PGP는 CA 없는 분산 신뢰 모델. S/MIME은 X.509 인증서 필요. IMAP은 이메일 수신 프로토콜(143번 포트).
📄 2019 국가직 정보시스템보안 문1 · HTTPS 구성 요소★★★
HTTPS를 구성하기 위해 필요한 프로토콜만을 모두 고르면?
ㄱ. TCP ㄴ. SSL ㄷ. SOAP ㄹ. SET
① ㄱ, ㄴ
② ㄱ, ㄹ
③ ㄴ, ㄷ
④ ㄷ, ㄹ
정답 ① — HTTPS = HTTP + SSL/TLS + TCP. SOAP은 웹 서비스 프로토콜, SET은 전자상거래 보안 프로토콜로 HTTPS 구성 요소가 아님.
📄 2020 국가직 정보시스템보안 문15 · CSRF 오답★★★
CSRF(Cross Site Request Forgery)의 특징에 대한 설명으로 옳지 않은 것은?
① 사용자가 자신의 의지와는 무관하게 공격자가 의도한 행위를 특정 웹사이트에 요청하게 하는 방법이다.
② 사이트에 방문하는 사용자가 정상적인 요청이 아닌 임의의 요청을 하도록 위조하는 방법이다.
③ 서버와 클라이언트에 각각 잘못된 시퀀스 넘버를 사용하여 연결된 세션에 혼란을 줌으로써 공격자가 서버와의 연결을 획득하는 방법이다.
④ 특정 웹사이트가 사용자의 웹브라우저를 신용하는 상태를 악용하는 방법이다.
정답 ③ — ③은 TCP 세션 하이재킹 설명. CSRF는 시퀀스 번호와 무관하며, 로그인된 사용자의 세션/쿠키를 이용해 의도치 않은 요청을 전송하는 공격.
📄 2020 국가직 정보시스템보안 문13 · 쿠키 오답★★
Cookie에 대한 설명으로 옳지 않은 것은?
① 쿠키는 클라이언트의 컴퓨터에 생성된다.
② 사용자와 웹사이트를 연결해 주는 정보가 저장되어 있다.
③ 직접 바이러스를 옮기거나 악성코드를 설치할 수 없다.
④ 사용자들의 로그인, 로그아웃, 시스템 재부팅 정보를 저장한다.
정답 ④ — 쿠키 용도: 사이트 개인화·장바구니·이용 방식 추적. 로그인·로그아웃·재부팅 정보는 시스템 로그(wtmp, utmp)에 저장. 쿠키는 4KB 이하 텍스트 파일로 악성코드 전파 불가.
📄 2020 국가직 정보시스템보안 문17 · SQL Injection 오답★★
SQL 인젝션에 대한 설명으로 옳지 않은 것은?
① 공격자가 입력값을 조작하여 원하는 SQL 구문을 실행한다.
② 전송되는 패킷을 가로채어 자신이 송신자인 것처럼 패킷을 변경하여 다시 보내는 기법이다.
③ 대응 방법 중에는 사용자의 입력에 특수문자가 포함되어 있는지 검증하는 방법이 있다.
④ OWASP에서 선정한 10대 웹 어플리케이션 보안 위험에 포함된다.
정답 ② — ②는 IP 스푸핑(IP Spoofing) 또는 패킷 변조 공격 설명. SQL Injection은 입력 폼에 SQL 코드 삽입하여 DB 조작하는 공격이며 패킷 가로채기와 관계없음.
📄 2020 국가직 정보시스템보안 문20 · XSS 공격 순서★★★
일반적인 XSS 공격 수행 과정을 순서대로 바르게 나열한 것은?
가. 사용자가 XSS 코드에 접근한다 / 나. 사용자 시스템에서 XSS 코드 실행 / 다. 서버가 XSS 포함 글 사용자에게 전달 / 라. 서버에 XSS 코드 작성·저장 / 마. 결과가 공격자에게 전달
① 라→가→나→다→마
② 라→가→다→나→마
③ 라→다→가→나→마
④ 라→다→나→가→마
정답 ③ — XSS 순서: 라(공격자 서버에 XSS 저장) → 다(서버가 XSS 포함 글 전달) → 가(사용자 접근) → 나(사용자 시스템에서 실행) → 마(결과 공격자에게). 서버가 먼저 전달(다)한 후 사용자가 접근(가)하는 순서 주의!
📄 2020 국회직 9급 · OWASP Top10 해당 안 되는 것★★
OWASP 2020에서 발표된 10가지 보안 위협에 속하지 않는 것은?
① Abuse of Cloud Computing
② Injection
③ XML External Entities
④ Broken Authentication
⑤ Sensitive Data Exposure
정답 ① — "Abuse of Cloud Computing(클라우드 컴퓨팅 남용)"은 OWASP Top 10에 없는 항목. ②Injection·③XML External Entities·④Broken Authentication·⑤Sensitive Data Exposure는 모두 2017/2020 OWASP Top 10 포함.
📄 2020 국회직 9급 · XSS 설명 (Stored XSS)★★★
게시판의 글에 원본과 함께 악성 코드를 삽입하여 글을 읽을 경우 악성코드가 실행되도록 하여 클라이언트의 정보를 유출하는 공격기법이다. 웹 페이지가 사용자로부터 입력받은 데이터를 필터링하지 않고 그대로 동적으로 생성된 웹 페이지에 포함하여 사용자에게 재전송할 때 발생한다.
① SQL Injection
② XSS (Cross Site Scripting)
③ 파일 업로드 취약점
④ CSRF (Cross Site Request Forgery)
⑤ 쿠키/세션 위조
정답 ② — "게시판 글에 악성 코드 삽입 + 읽을 때 실행 + 입력값 필터링 없이 재전송" = Stored XSS. CSRF는 사용자가 모르게 요청을 서버에 전송하는 공격으로 스크립트 실행이 주목적 아님.
📄 2021 군무원 7급 · 이메일 보안 기술★★★
이메일 보안을 위해 사용되는 보안 기술로만 묶인 것으로 가장 옳은 것은?
① SET, S/MIME
② PEM, PGP
③ S/MIME, IDS
④ PGP, SSO
정답 ② — 전자우편 보안 기술: PGP, S/MIME, PEM. SET은 전자상거래(신용카드) 보안, IDS는 침입탐지시스템, SSO는 단일 로그인 인증. ①의 SET·③의 IDS·④의 SSO는 이메일 보안 기술이 아님.

XSS vs CSRF — 핵심 차이 한눈에
💡 8단원 핵심 암기
XSS클라이언트 브라우저에서 스크립트 실행. 서버가 입력값 신뢰. 세션탈취·페이지변조.
CSRF사용자 모르게 서버에 요청 전송. 서버가 사용자 브라우저 신뢰. 송금·수정 강제.
SQL InjectionSQL 코드 삽입 → DB 인증 우회. 대응: Prepared Statement, 특수문자 필터링.
XSS 순서라(저장) → 다(서버 전달) → 가(사용자 접근) → 나(실행) → 마(결과 전달)
HTTPSHTTP+SSL. 포트 443. URL·문서내용·쿠키·HTTP헤더 암호화. RFC 2818.
쿠키클라이언트에 저장. 4KB 이하 텍스트. 바이러스 전파 불가. 개인화·장바구니 용도.
PGP1991, 필 짐머만. CA 없음, 무료. IDEA+RSA+SHA-1. 분산 신뢰 모델.
S/MIMERSA 개발. X.509 v3 인증서 사용. Triple-DES(메시지) + DSA(서명).
PEMIETF 표준. Base64 인코딩. PGP보다 보안능력 뛰어남. 중앙집중식 어려움.
이메일 보안PGP + S/MIME + PEM. SET·IDS·SSO는 이메일 보안 기술 아님.

  • "SQL Injection은 패킷을 가로채 변경하는 기법" → ✗ (IP 스푸핑)
  • "쿠키는 서버에 저장되며 로그인/재부팅 정보를 담는다" → ✗
  • "PGP는 CA로부터 X.509 인증서를 발급받아야 한다" → ✗ (S/MIME이 X.509 사용)
  • "HTTPS 사용 시 IP 헤더의 TTL 필드도 암호화된다" → ✗ (TTL은 암호화 안 됨)
  • "Abuse of Cloud Computing이 OWASP Top 10에 포함된다" → ✗
  • "XSS = 클라이언트 브라우저에서 스크립트 실행" → ✓
  • "CSRF = 사용자 의지와 무관한 서버 요청 강제" → ✓
  • "HTTPS 암호화: URL·문서내용·쿠키·HTTP헤더" → ✓
  • "PGP = CA 없음, 무료, 1991년, 필 짐머만" → ✓
  • "전자우편 보안 = PGP·S/MIME·PEM (SET·IDS·SSO 아님)" → ✓
